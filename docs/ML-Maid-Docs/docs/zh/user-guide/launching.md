# 启动与游玩记录

至少添加一个“文件”动作并选择存在的可执行文件。工作目录为空时，使用可执行文件所在目录。启动成功后按钮显示“游戏中”，停止前不能重复启动。

## 监控模式

| 模式 | 行为 | 适用情况 |
| --- | --- | --- |
| 文件模式 | 只等待直接启动的进程 | 游戏 EXE 本身持续运行 |
| 文件夹模式 | 监控工作目录内的 EXE | 默认推荐；多数视觉小说 |
| 进程模式 | 按配置的进程名查找进程 | 游戏进程位于其他目录 |

直接进程运行至少 10 秒时记为有效会话。若其在 10 秒内退出，系统将其视为启动器，等待 5 秒后寻找真实游戏进程，并每 3 秒检查一次。找不到目标进程时不累计时长；单次监控最多持续 12 小时。有效会话结束后会更新累计时长、上次游玩和统计数据。

## 日文转区

- 关闭：直接启动。
- Locale Emulator：通过 `LEProc.exe <游戏.exe>` 启动，适合旧式日文引擎。
- 基础模式：设置 `LANG` 和 `LC_ALL` 为 `ja_JP.UTF-8`，只对读取这些变量的程序有效。

使用 Locale Emulator 前，在“设置 → 启动器”自动检测或手动选择 `LEProc.exe`。路径无效时不会创建会话记录。
